# AGENTS.md

- 기능은 작고 검증 가능한 단위로 구현한다.
- 구현이 작더라도 아키텍처 경계는 깨끗하게 유지한다.
- 현재 기능이 요구하지 않는 시스템은 미리 설계하지 않는다.
- 인프라, 배포, 데이터베이스, 원격 서비스 변경 전에는 먼저 확인한다.
- 터미널/CLI 판타지 RPG 스타일을 유지한다.
- Supabase는 서버 권위 백엔드로 취급한다.
- DB 스키마, Edge Functions, 서버 로직은 현재 기능이 필요로 할 때만 추가한다.
- 새 프레임워크나 추상화보다 기존 프로젝트 패턴을 우선한다.
- 앱 안의 문구는 기능적으로 짧게 유지하고, 설명문/마케팅 문구를 피한다.
- 프론트 변경 후에는 빌드를 기본 검증으로 삼고, 브라우저 확인은 시각/상호작용 검증이 꼭 필요할 때만 수행한다.
