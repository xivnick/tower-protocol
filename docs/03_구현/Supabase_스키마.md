# Supabase 스키마

Supabase는 원격 서버 권위 구조의 기반으로 사용한다.
다만 현재 단계에서는 기능을 크게 앞서 설계하지 않고, 실제로 필요한 구조만 작게 추가한다.

## 현재 원칙

- 먼저 로그인과 프로필만 둔다.
- 캐릭터, 인벤토리, 전투, 정수, 밸런스 운영 테이블은 아직 만들지 않는다.
- 새 기능을 구현할 때 해당 기능에 필요한 최소 테이블과 정책만 추가한다.
- 클라이언트가 보상, 전투 결과, 랭킹 같은 신뢰 데이터 값을 직접 쓰게 하지 않는다.

## 현재 마이그레이션

원격 프로젝트에는 과거 실험 마이그레이션이 적용된 이력이 있다.
현재 유효 스키마는 reset 마이그레이션으로 최소화했다.

```txt
supabase/migrations/003_reset_to_minimal_schema.sql
```

현재 남기는 테이블:

```txt
profiles
```

## profiles

사용자 프로필의 최소 정보만 저장한다.

```txt
user_id
nickname
created_at
updated_at
```

권한:

- 로그인한 사용자는 자기 프로필만 읽을 수 있다.
- 로그인한 사용자는 자기 프로필만 생성할 수 있다.
- 로그인한 사용자는 자기 프로필만 수정할 수 있다.

## 다음에 추가할 후보

기능 구현 순서에 따라 필요한 시점에만 추가한다.

```txt
characters
inventory_items
equipped_slots
hunt_runs
tower_progress
```

각 테이블은 UI/시스템 구현이 실제로 필요해지는 순간에 설계한다.
