# 데이터 구조

기존 구현 기준 파일은 `rpg-tower-data.js`다.
새 100층 정수 목록과 밸런스 검토 기준은 `04_게임_데이터/01_정수_목록.md`와 `rpg-tower-balance-sim.mjs`다.
배포 후 밸런스 운영 데이터 구조는 [[운영_밸런스_데이터]]를 따른다.
Supabase DB 스키마는 [[Supabase_스키마]]에 둔다.

## 전역 데이터

`rpg-tower-data.js`는 브라우저와 Node 시뮬레이터에서 함께 쓰기 위해 데이터를 전역 객체에 붙이는 구조다.

주요 데이터:

- `GAME_RULES`
- `ACCESSORIES`
- `ESSENCES`
- `MONSTERS`
- `HUNTING_AREAS`
- `TOWER_FLOORS`

## GAME_RULES

게임 공통 규칙 상수.

```js
const GAME_RULES = {
  recoveryActionSeconds: 1,
};
```

`recoveryActionSeconds`는 휴식/포션 같은 회복 액션이 소모하는 시간이다.

현재 규칙:

- 휴식은 이 시간만큼 소요된다.
- 휴식 후 체력은 100%가 된다.
- 휴식 중 액티브 정수 쿨타임도 이 시간만큼 감소한다.
- 휴식 실행 시점에 포션을 보유 중이면 포션을 자동 사용한다.
- 포션 사용은 휴식을 한 것으로 취급하므로 같은 시간 소모와 쿨타임 감소를 적용한다.

## ACCESSORIES

장신구 데이터.

각 장신구는 키와 이름, 효과 필드를 가진다.

예:

```js
giantHeart: { name: "거인의 심장", hpPct: 0.18 }
hawkEye: { name: "매의 눈", accPct: 0.18 }
precisionScope: { name: "정밀한 조준경", overcritToCritDamage: 0.5 }
```

## ESSENCES

정수 데이터.

`rpg-tower-data.js`의 `ESSENCES`는 구버전 구현 데이터다.
현재 100층 정수 목록은 [[04_게임_데이터/01_정수_목록|정수 목록]] 문서를 기준으로 검토한다.
새 설계가 확정되면 `ESSENCES`를 문서 기준 데이터 구조로 교체한다.

기존 정수는 다음 구조를 가진다.

```js
{
  name: "성난 멧돼지의 괴력",
  type: "active",
  cooldown: 5,
  grades: {
    1: { ... },
    2: { ... },
    3: { ... },
    4: { ... },
    5: { ... }
  }
}
```

패시브 정수는 `cooldown`이 없을 수 있다.

최신 검증 기준:

```txt
탑 1~100층 정수 효과 100개
탑 몬스터가 사용하는 4등급 효과 100개
문서 파싱 누락 0개
```

## MONSTERS

몬스터 데이터.

각 몬스터는 레벨, 지역, 이름, 연결 정수, 스탯/전투 성향을 가진다.

시뮬레이터 목록 기준으로 몬스터는 1~100레벨 구간까지 정의되어 있다.

## HUNTING_AREAS

사냥터 데이터.

현재 20개 사냥터를 자동 생성한다.

```js
const HUNTING_AREAS = Array.from({ length: 20 }, ...)
```

각 사냥터는 5레벨 구간을 담당한다.

```txt
hunt01: 1~5
hunt02: 6~10
...
hunt20: 96~100
```

## TOWER_FLOORS

탑 데이터.

현재 100층을 자동 생성한다.

```js
const TOWER_FLOORS = Array.from({ length: 100 }, ...)
```

탑 구성:

```txt
각 5층 구간의 1~4층: elite
각 5층 구간의 5층: boss
```

탑 레벨은 `towerEnemyLevel(floor)`로 계산한다.

## 탑 구간 데이터

확정 규칙상 100층 탑은 20개 사냥터와 대응하는 20개 구간이다.

```txt
사냥터 1 -> 탑 1~5층
사냥터 2 -> 탑 6~10층
...
사냥터 20 -> 탑 96~100층
```

각 구간에는 일반 몬스터 4종과 보스 1종이 필요하다.

현재 `rpg-tower-data.js`는 `TOWER_SEGMENT_MONSTERS`로 20개 구간을 명시한다.

각 `HUNTING_AREAS` 항목은 다음 필드를 가진다.

```txt
monsters: 사냥터 일반 몬스터 4종
boss: 해당 탑 구간 보스
```

각 `TOWER_FLOORS` 항목은 해당 사냥터 구간의 1~4번째 몬스터를 elite로, 5번째 몬스터를 boss로 배치한다.

## 구현 파일

- `rpg-tower-data.js`: 공용 데이터

## 운영 데이터 전환

현재 `rpg-tower-data.js`는 로컬 검토용 공용 데이터다.
실제 운영 단계에서는 이 파일을 직접 수정하지 않고, 어드민에서 관리하는 밸런스 버전 데이터를 사용한다.

전환 목표:

```txt
rpg-tower-data.js 임시 데이터
-> seed JSON
-> balance_entities 개별 편집 데이터
-> compiled balance snapshot
-> 게임 서버 active balance cache
```

자세한 기준은 [[운영_밸런스_데이터]]에 둔다.
DB 마이그레이션은 [[Supabase_스키마]]를 따른다.

현재 seed 생성:

```bash
node scripts/generate-balance-seed.mjs
node scripts/generate-unparsed-effects-report.mjs
```

출력:

```txt
data/balance/seed/balance-seed.v0.1.0.json
data/balance/seed/unparsed-effects-report.md
```

복잡한 트리거/스택/상태 효과는 자동 파싱 후 수동 override를 적용한다.

```txt
data/balance/manual-effect-overrides.json
```
