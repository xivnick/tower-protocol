# 문서 작성 원칙

이 문서는 `TOWER://` 기획 문서를 정리할 때 따르는 기준이다.

## 기본 기준

- 최신 기획 문서는 실제 게임 규칙과 플레이 경험을 기준으로 작성한다.
- 한 문서에는 하나의 역할만 둔다.
- 같은 내용을 여러 문서에 중복해서 유지하지 않는다.
- 기준이 바뀌면 최신 문서를 먼저 고친다.
- 이전 기준이나 폐기한 내용은 `99_보관`으로 내린다.

## 기획 문서

기획 문서는 게임 안에서 플레이어가 접하는 내용을 설명한다.

포함하는 내용:

- 플레이어가 하는 행동
- 게임 안에서 적용되는 규칙
- 콘텐츠의 목적과 보상
- 성장 방식과 선택지
- 확정된 수치, 조건, 제한

표현 기준:

- 문장은 짧게 쓴다.
- 개요 문서는 명사형이나 짧은 설명 위주로 쓴다.
- 세부 규칙 문서는 조건, 처리 순서, 예외를 분명히 쓴다.
- 아직 정하지 않은 내용은 확정처럼 쓰지 않는다.
- 논의 기록과 작업 로그는 최신 기획 문서와 분리한다.

## 구현 문서

구현 문서는 실제 개발과 운영에 필요한 내용을 다룬다.

포함하는 내용:

- 데이터 구조
- DB 스키마
- 시뮬레이터와 검증 방법
- 배포 후 밸런스 운영 방식
- 어드민에서 수정하는 데이터의 흐름

## 보관 문서

보관 문서는 이전 결정과 초안을 남겨두는 곳이다.
보관 문서는 최신 기준이 아니며, 필요한 경우 근거 확인용으로만 사용한다.

## 링크와 파일명

- 폴더와 핵심 문서는 읽는 순서가 보이도록 번호를 붙인다.
- 문서 제목은 실제 역할을 드러내는 이름으로 쓴다.
- 문서 이름이 바뀌면 홈과 관련 링크를 함께 수정한다.
