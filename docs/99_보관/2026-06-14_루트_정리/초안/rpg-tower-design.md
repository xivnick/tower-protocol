# 웹 RPG 설계 문서

## 기본 컨셉

- 장르: 웹 기반 자동사냥 RPG
- 핵심 루프: 사냥 → 장비·정수·골드 획득 → 캐릭터 강화 → 탑 도전 → 랭킹
- 랭킹 기준: 최고 도달 층수 + 먼저 클리어한 시점 (동률 시 선점 우선)
- MVP 범위: 사냥, 장비, 정수, 스탯 세팅, 수동 탑 도전에 집중
- MVP 제외: PvP, 거래, 길드, 시즌 리셋

## 스탯 시스템

### 1차 스탯 (레벨업 시 직접 분배)

| 스탯 | 영문 | 설명 |
|------|------|------|
| 근력 | STR | 물리 공격력에 영향 |
| 민첩 | AGI | 공격속도, 회피에 영향 |
| 손재주 | DEX | 치명타 확률, 치명타 피해, 명중에 영향 |
| 체력 | VIT | 최대 체력, 물리 방어에 영향 |
| 인내 | END | 방어력, 체력 재생에 영향 |
| 지력 | INT | 스킬 위력에 영향 |
| 지혜 | WIS | 스킬 쿨타임 감소에 영향 |

### 2차 스탯 (1차 스탯 연산)

| 2차 스탯 | 기반 스탯 | 설명 |
|----------|----------|------|
| 물리 공격력 | STR | 기본 공격 피해 |
| 공격속도 | AGI | 턴당 행동 빈도 |
| 치명타 확률 | DEX | 치명타 발생 확률 |
| 치명타 피해 | DEX | 치명타 발생 시 피해 배율 |
| 명중 | DEX | 상대 회피 상쇄 |
| 스킬 위력 | INT | 정수 스킬 효과 수치 |
| 최대 체력 | VIT | HP 최대치 |
| 물리 방어 | VIT + END | 물리 피해 감소 |
| 마법 방어 | END + WIS | 마법 피해 감소 |
| 최종 방어력 | 물리 방어 + 마법 방어 | 실제 피해 감소에 사용 |
| 체력 재생 | END | 매 턴 HP 회복량 |
| 회피 | AGI | 공격 빗나감 확률 계산에 사용 |
| 스킬 쿨타임 감소 | WIS | 전체 스킬 쿨타임 감소율 |

### 빌드 방향

두 개 스탯을 집중 육성하는 게 기본 방향 (1:1 또는 2:1 비율).
나머지 스탯에도 최소 투자 필요 — 딜×생존 곱셈 구조라 한쪽만 몰빵하면 탑에서 버티지 못함.
스탯 재분배는 가능하며, 비싸지 않은 가격의 재화를 소모한다.

주요 빌드 조합 예시:

- STR + VIT: 안정적인 물리 딜탱
- AGI + DEX: 빠르고 치명타 잘 나오는 암살자
- INT + WIS: 스킬 위력과 쿨타임 모두 챙기는 스킬 빌드
- AGI + WIS: 빠른 행동속도로 스킬을 자주 쓰는 빌드
- VIT + END: 방어와 체력 재생을 극대화한 탱커

각 빌드는 혼자서 몬스터를 처치할 수 있는 공격 플랜을 가져야 한다.
파티 RPG가 아니므로 방어형 빌드도 반사 피해, 방어력 기반 피해, 재생 연계 피해처럼 적을 죽이는 수단이 필요하다.

빌드별 공격 플랜:

| 빌드 | 생존 방식 | 처치 방식 |
|------|----------|----------|
| 근력 중심 | 체력, 방어 일부 | 강타, 괴력, 강한 물리 피해 |
| 민첩 + 손재주 | 회피, 빠른 행동 | 연격, 치명타, 타격당 고정 피해 |
| 지력 + 지혜 | 마법 방어, 쿨타임 운용 | 마법 피해, 지속 피해, 스택형 정수 |
| 체력 + 인내 | 높은 체력, 방어, 재생 | 반사 피해, 방어력 기반 피해 |
| 인내 + 지혜 | 방어, 재생, 약화 | 디버프 누적, 재생/반사 연계 피해 |

지혜는 직접 피해를 크게 올리는 스탯이 아니라 정수 쿨타임 감소를 통해 전투 효과를 빠르게 누적하는 운용형 스탯이다.
스택형 정수는 지혜 빌드의 핵심 대상이며, 여러 번 사용할수록 피해, 약화, 회복, 강화 효과가 커진다.

### 스탯 및 효과 적용 순서

스탯 계산은 1차 스탯 → 2차 스탯 → 전투 결과 순서로 처리한다.
고정값 증가와 퍼센트 증가는 서로 다른 계층으로 구분한다.

기본 계산 순서:
1. 기본 1차 스탯
2. 레벨업으로 분배한 1차 스탯
3. 장비, 정수, 보석, 버프의 1차 스탯 고정 증가
4. 장비, 정수, 보석, 버프의 1차 스탯 퍼센트 증가
5. 최종 1차 스탯 확정
6. 최종 1차 스탯을 기반으로 2차 스탯 계산
7. 장비, 정수, 보석, 버프의 2차 스탯 고정 증가
8. 장비, 정수, 보석, 버프의 2차 스탯 퍼센트 증가
9. 최종 2차 스탯 확정
10. 전투 공식 적용
11. 최종 피해, 회복, 쿨타임 등 결과값 보정
12. 상한/하한 적용

1차 스탯 계산:

```
최종 1차 스탯 = (기본 1차 스탯 + 분배 스탯 + 1차 스탯 고정 보너스) × (1 + 1차 스탯 퍼센트 보너스)
```

2차 스탯 계산:

```
기본 2차 스탯 = 최종 1차 스탯 기반 공식
최종 2차 스탯 = (기본 2차 스탯 + 2차 스탯 고정 보너스) × (1 + 2차 스탯 퍼센트 보너스)
```

전투 결과 보정:

```
최종 결과값 = 전투 공식 결과 × (1 + 최종 배율 보너스) + 최종 고정 보너스
```

퍼센트 보너스는 같은 계층 안에서 합산한다.

예시:

```
STR +20%
STR +15%

적용 배율 = 1 + 0.20 + 0.15 = 1.35
```

즉, 같은 계층의 퍼센트 보너스는 `×1.20×1.15`가 아니라 `×1.35`로 처리한다.

효과 계층 예시:

| 계층 | 예시 |
|------|------|
| 1차 스탯 고정 증가 | STR +20, AGI +10 |
| 1차 스탯 퍼센트 증가 | STR +20%, WIS +15% |
| 2차 스탯 고정 증가 | 물리 공격력 +50, 회피 +30 |
| 2차 스탯 퍼센트 증가 | 물리 공격력 +30%, 회피 +20% |
| 최종 배율 보너스 | 최종 피해 +20%, 회복량 +15%, 받는 피해 -10% |
| 최종 고정 보너스 | 피해 +10, 회복 +20 |

계산 예시:

```
기본 STR 100
장비 STR +20
정수 STR +10
버프 STR +20%

최종 STR = (100 + 20 + 10) × 1.20 = 156
```

STR이 물리 공격력으로 1당 2씩 변환된다고 가정:

```
기본 물리 공격력 = 156 × 2 = 312
무기 물리 공격력 +50
버프 물리 공격력 +30%

최종 물리 공격력 = (312 + 50) × 1.30 = 470.6
```

최종 배율 보너스는 치명타, 취약, 보스 추가 피해, HP 조건부 피해 증가처럼 전투 결과 자체를 바꾸는 특수 효과에 제한적으로 사용한다.
최종 배율 계층을 많이 쓰면 수치가 쉽게 폭주하므로, 일반 장비 옵션은 가급적 1차 또는 2차 스탯 보너스로 설계한다.

## 전투 계산식

### 피해 계산

```
최종 피해 = 공격력 × (100 / (100 + 방어력))
```

방어력에 따른 방어율:

| 방어력 | 방어율 |
|--------|--------|
| 0 | 0% |
| 50 | 33% |
| 100 | 50% |
| 200 | 67% |
| 300 | 75% |
| 500 | 83% |
| 1000 | 91% |

### 치명타 계산

```
치명타 피해 = 최종 피해 × 치명타 피해율
```

- 치명타 피해율 기본값: 150%
- DEX로 치명타 확률과 치명타 피해율 상승

### 고정 피해

고정 피해는 방어력 계산과 치명타 계산이 끝난 뒤, 각 타격마다 마지막에 더해지는 방어 무시 피해다.

```
최종 피해 = 치명타까지 계산된 피해 + 고정 피해
```

- 방어력의 영향을 받지 않음
- 치명타 피해율의 영향을 받지 않음
- 타격마다 적용되므로 연격과 시너지가 큼
- 고정 피해는 공격력에 더하는 수치가 아니라, 피해 계산 이후 더하는 별도 피해

예시:

```
기본 피해: 100
방어력 적용 후 피해: 60
치명타 적용 후 피해: 90
고정 피해: 10

최종 피해: 100
```

고정 피해 정수는 방어를 무시하고 틈을 찌르는 느낌을 주는 정수에 사용한다.
예: 날붙이, 관통침, 약점 포착, 톱니 이빨

### 빗나감 계산

```
빗나감 확률 = 회피 / (회피 + 명중)
```

- 회피 기본값: 0 (투자하지 않으면 무조건 맞음)
- 명중 기본값: 100부터 시작
- 플레이어에게는 명중 100 기준의 빗나감 확률로 표시

명중 100 기준, 회피 수치에 따른 빗나감 확률:

| 회피 | 빗나감 확률 |
|------|------------|
| 0 | 0% |
| 10 | 9% |
| 30 | 23% |
| 50 | 33% |
| 100 | 50% |
| 200 | 67% |
| 300 | 75% |
| 500 | 83% |

### 회피 옵션 종류

- 회피 +N: 회피 수치를 N만큼 올림
- 회피 +N%: 현재 회피 수치에 N%를 곱해서 추가

회피 100, 명중 100 기준 옵션별 효과:

| 옵션 | 회피 수치 | 빗나감 확률 | 증가량 |
|------|----------|------------|--------|
| 기본 | 100 | 50% | - |
| 회피 +10 | 110 | 52% | +2% |
| 회피 +30 | 130 | 57% | +7% |
| 회피 +50% | 150 | 60% | +10% |
| 회피 +100% | 200 | 67% | +17% |

## 전투 시스템

서버에서 처리하는 틱 기반 전투 시뮬레이션.
전투는 턴 기반 감각을 유지하되, 실제 시간 흐름은 틱 단위로 계산한다.

기본 시간 단위:
- 10틱 = 1초
- 공격속도가 높을수록 일반공격 게이지가 빠르게 차오름
- 내부 계산은 소수점 틱을 허용

### 공격속도 계산

공격속도는 하드캡 대신 소프트캡을 사용한다.
기본 공격속도 100은 초당 1회 공격을 의미한다.

```
초당 공격 횟수 = sqrt(최종 공격속도 / 100)
틱당 일반공격 게이지 증가량 = 초당 공격 횟수 / 10
```

전투 내부에서는 매 틱마다 일반공격 게이지를 누적하고, 게이지가 1 이상이면 일반공격을 실행한다.

```
매 틱:
일반공격 게이지 += 틱당 일반공격 게이지 증가량

일반공격 게이지 >= 1.0 이면:
일반공격 실행
일반공격 게이지 -= 1.0
```

공격속도 예시:

| 최종 공격속도 | 초당 공격 횟수 | 틱당 게이지 증가 | 평균 공격 간격 |
|---------------|----------------|------------------|----------------|
| 25 | 0.50회 | 0.0500 | 20틱 (2.00초) |
| 50 | 0.71회 | 0.0707 | 14.14틱 (1.41초) |
| 100 | 1.00회 | 0.1000 | 10틱 (1.00초) |
| 150 | 1.22회 | 0.1225 | 8.16틱 (0.82초) |
| 200 | 1.41회 | 0.1414 | 7.07틱 (0.71초) |
| 400 | 2.00회 | 0.2000 | 5틱 (0.50초) |
| 900 | 3.00회 | 0.3000 | 3.33틱 (0.33초) |

일반공격 게이지는 소수점으로 유지한다.
공격속도 50처럼 평균 공격 간격이 14.14틱인 경우 실제 공격은 14틱/15틱 간격이 섞이며 평균값에 수렴한다.

### 일반공격 속도 보정

민첩, 장비, 패시브 정수는 공격속도 수치를 올린다.
둔화, 가속, 보스 패턴처럼 전투 중 속도를 바꾸는 효과는 공격속도 수치를 직접 바꾸지 않고 일반공격 속도 배율로 적용한다.

```
기본 틱당 일반공격 게이지 = sqrt(최종 공격속도 / 100) / 10
최종 틱당 일반공격 게이지 = 기본 틱당 일반공격 게이지 × 일반공격 속도 배율
```

일반공격 속도 보정은 곱연산으로 중첩한다.

예:

```
20% 둔화 1개: 일반공격 속도 배율 0.8
20% 둔화 2개: 일반공격 속도 배율 0.8 × 0.8 = 0.64
```

공격속도 100 기준:

| 상태 | 틱당 게이지 | 평균 공격 간격 | 초당 공격 |
|------|-------------|----------------|-----------|
| 기본 | 0.100 | 10.00틱 | 1.00회 |
| 20% 둔화 1개 | 0.080 | 12.50틱 | 0.80회 |
| 20% 둔화 2개 | 0.064 | 15.63틱 | 0.64회 |

같은 시점에 여러 행동이 겹치면 다음 우선순위로 처리:
1. 플레이어 정수
2. 플레이어 일반공격
3. 몬스터 정수
4. 몬스터 일반공격

정수 스킬은 일반공격과 별도의 이벤트로 처리되며, 쿨타임마다 자동 발동한다.
플레이어 정수 슬롯이 여러 개 동시에 발동 가능하면 슬롯 순서가 우선순위가 된다.

### 정수 쿨타임 시작 규칙

정수 쿨타임은 콘텐츠 성격에 따라 다르게 처리한다.

일반 자동사냥:
- 자동사냥은 연속된 시간 흐름으로 본다.
- 전투가 끝나도 액티브 정수의 남은 쿨타임은 다음 전투로 이어진다.
- 처음 자동사냥을 시작할 때는 장착한 액티브 정수가 준비된 상태로 시작한다.
- 이 방식은 짧은 전투마다 첫 발동만 반복되는 문제를 줄인다.

탑 도전:
- 탑은 수동 도전 콘텐츠이므로 전투 시작 시 장착한 액티브 정수가 준비된 상태로 시작한다.
- 보스전도 동일하게 준비된 상태로 시작한다.

액티브 정수는 너무 드물게 발동되지 않도록 주로 3~7초 쿨타임에 배치한다.
보스 정수도 긴 쿨타임의 강한 한 방이 아니라, 새로운 전투 스타일을 만드는 독특한 정수로 설계한다.
보스 정수는 액티브일 수도 있고, 패시브일 수도 있으며, 자주 발동되거나 조건부로 한 번 크게 개입할 수도 있다.

한쪽 HP가 0이 되면 전투 종료.
결과: 소모 턴 수, 생존 여부, 드랍 목록.
전투 결과는 플레이어에게 로그 형태로 요약 제공.

## 사냥

- 자동사냥
- 게임을 꺼둔 동안에도 자동사냥 진행
- 유저가 사냥터를 직접 선택
- 현재 세팅으로 무난한 사냥터 하나에 추천 태그 표시
- 초반에는 진입 가능한 사냥터 수를 적게 제한
- 특정 정수 필요 시 해당 몬스터 사냥터로 이동
- 죽으면 일정 시간 사망 패널티 후 자동 재개
- 드랍: 장비, 정수, 골드

### 회복 루틴

자동사냥은 전투, 회복 판단, 다음 전투가 이어지는 흐름으로 처리한다.

기본 회복 규칙:
- 공통 회복 액션 시간은 `RECOVERY_ACTION_SECONDS = 1`로 둔다.
- 자동사냥에서 전투가 끝나면 자동 사냥 설정에 따라 휴식 여부를 판단
- 휴식을 취하면 `RECOVERY_ACTION_SECONDS`만큼 시간을 소모하고 체력이 100%까지 회복됨
- 휴식 중 액티브 정수 쿨타임도 `RECOVERY_ACTION_SECONDS`만큼 감소
- 휴식 실행 시점에 포션을 보유 중이면 포션을 자동 사용
- 포션 사용은 휴식을 한 것으로 취급하며, 같은 시간 소모와 정수 쿨타임 감소가 적용됨
- 포션을 충분히 보유하면 위험한 사냥터에서도 휴식 기준에 걸린 뒤 바로 회복할 수 있어 사냥 효율을 높일 수 있음
- 사망 패널티는 기본 10초
- 사망 패널티 시간 동안 체력과 액티브 정수 쿨타임도 같은 시간만큼 회복

전투 종료 후 회복 판단:
- 전투 종료 후 체력이 설정한 기준 이하이면 휴식할 수 있음
- 휴식 기준은 유저가 설정하거나, 추천 사냥터에서는 기본값을 사용할 수 있음
- 예: 체력 50% 이하이면 `RECOVERY_ACTION_SECONDS`만큼 휴식 후 체력 100%로 다음 전투 진행

휴식은 자동사냥의 안정성을 올리지만 시간당 처치 수를 낮춘다.
체력, 인내, 재생 정수, 흡혈 정수는 휴식 빈도를 줄여 시간당 보상을 높이는 역할을 가진다.
포션은 휴식 타이밍에 자동 소모되어 회복 실패를 막고 사냥 효율을 높이는 소모품이다.

## 정수 시스템

정수는 몬스터가 실제 전투에서 사용하는 능력을 플레이어가 얻어 장착하는 시스템이다.
플레이어는 몬스터와 싸우며 해당 몬스터의 능력을 경험하고, 그 정수를 얻으면 같은 느낌의 능력을 사용할 수 있다.

정수 이름에는 몬스터 이름을 붙인다.

예:
- 성난 멧돼지의 괴력 정수
- 숲 늑대의 연격 정수
- 칼날 딱정벌레의 날붙이 정수

몬스터가 전투에서 사용하지 않는 정수는 드랍하지 않는다.
몬스터 버전과 플레이어 버전은 같은 콘셉트를 유지하되, 밸런스를 위해 계수와 쿨타임은 다르게 조정할 수 있다.

### 등급 구조

- 같은 정수 3개 합성 → 상위 등급 정수
- 등급 상승 시 스킬 효과 강화 또는 조건부 효과 추가
- 서로 다른 몬스터의 정수는 합성 재료로 섞지 않음
- 정수 강화가 상위 몬스터 정수로 변하지 않음

정수 등급:

| 등급 | 이름 | 감각 |
|------|------|------|
| 1등급 | 희미한 | 약한 맛보기 |
| 2등급 | 온전한 | 실사용 시작 |
| 3등급 | 응축된 | 빌드 핵심 가능 |
| 4등급 | 정제된 | 고성능 |
| 5등급 | 각성한 | 완성형 |

1등급은 처음 얻는 맛보기 성격이 강하고, 2등급부터 실제 세팅에 사용할 만한 성능을 가진다.

드랍 등급:

| 출처 | 드랍 |
|------|------|
| 일반 사냥터 몬스터 | 해당 몬스터 정수 1등급 |
| 탑 엘리트 몬스터 | 같은 몬스터 정수 1~3등급 |
| 탑 보스 | 보스 고유 정수 1~3등급 |

탑 정수 등급 확률:

| 등급 | 확률 |
|------|------|
| 1등급 | 50% |
| 2등급 | 30% |
| 3등급 | 20% |

탑 엘리트 몬스터의 정수 드랍률은 10%다.

### 보스 정수

- 계열에 속하지 않는 고유 정수
- 보스 클리어 시 획득
- 최초 클리어 시 보스 정수 1개 확정
- 이후 재입장 클리어 시 확률 드랍
- 드랍되는 보스 정수의 등급은 1~3등급 중 랜덤
- 일반 정수보다 독특한 콘셉트를 가짐
- 긴 쿨타임의 강한 한 방보다 새로운 빌드 방향이나 전투 스타일을 제안
- 구간에서 배운 정수 효과를 응용하거나, 다른 정수 조합을 유도

### 스킬 슬롯

- 총 3개 슬롯
- 액티브 정수와 패시브 정수를 구분하지만, 장착 슬롯은 공유
- 액티브 정수는 쿨타임 기반 자동 발동
- 패시브 정수는 장착 중 항상 적용
- 액티브 정수끼리 동시에 발동 가능하면 슬롯 순서가 우선순위
- WIS: 전체 쿨타임 감소 담당

정수 타입:

| 타입 | 설명 | 예시 |
|------|------|------|
| 액티브 | 쿨타임마다 자동 발동 | 강타, 연격, 괴력, 재생, 독포자 |
| 패시브 | 장착 중 항상 적용 | 체력 재생 증가, 회피 증가, 고정 피해 추가, 방어력 증가 |

3개 슬롯을 어떻게 구성하느냐에 따라 전투 방식이 달라진다.

예:
- 3 액티브: 정수 스킬을 많이 쓰는 빌드
- 2 액티브 + 1 패시브: 일반적인 안정형
- 1 액티브 + 2 패시브: 기본공격/스탯 강화형
- 3 패시브: 단순하지만 꾸준한 빌드

### 정수 효과 방향

초반 정수는 몬스터의 신체 특징이나 전투 습성을 단순하게 가져오는 방향으로 설계한다.

예:
- 강타: 강한 1회 물리 피해
- 괴력: 일정 시간 물리 공격력 증가
- 연격: 약한 여러 번 물리 피해
- 돌가죽: 일정 시간 방어력 증가
- 재생: 체력 재생 증가 또는 회복
- 흡혈: 피해를 주고 일부 회복
- 날붙이: 타격마다 고정 피해 추가

방어형 빌드를 위해 다음 효과도 정수로 제공한다.

- 반사 피해: 받은 피해의 일부를 되돌림
- 고정 반사 피해: 피격될 때마다 고정 피해를 줌
- 방어력 기반 피해: 자신의 방어력 일부를 피해로 전환
- 재생 연계 피해: 회복 또는 재생이 발생할 때 적에게 피해

지혜 빌드를 위해 스택형 정수를 제공한다.

- 정수를 사용할 때마다 스택을 쌓음
- 스택 수에 따라 피해, 약화, 회복, 강화 효과 증가
- 일정 스택 도달 시 추가 효과 발동

### 보석 시스템

- 보석 시스템은 보류 기능이며 추후 업데이트 예정
- 아래 내용은 과거 초안
- 일정 등급 이상 정수에 보석 슬롯 해금
- 등급 높을수록 슬롯 수 증가 (3등급 1개, 4등급 2개, 최고 등급 3개)

보석 종류:
- 쿨타임 감소: 해당 스킬 쿨타임 단축
- 쿨타임 증가 + 효과 강화: 쿨타임을 늘리는 대신 효과 대폭 증가
- 조건부 강화: 특정 조건 충족 시 효과 강화 (예: HP 50% 이하일 때 위력 증가)

## 탑

### 기본 구조

- 순차 해금: 아래층 클리어 후 위층 입장 가능
- 탑 도전은 유저가 수동으로 시작
- 층마다 소액 골드를 내고 입장
- 실패 시 골드 환급 없음
- 클리어한 층은 언제든 재입장 가능 (보스 정수 파밍 목적)

### 구간별 입장 비용

탑 입장 비용은 실제 게임 규칙이다.
층별 비용 공식은 아직 정하지 않았다.

과거 방향:
- 1-10층: 저렴
- 11-20층: 중간
- 21-30층: 비쌈

### 층 구성

- 사냥터 하나가 탑 5층짜리 구간 하나와 연결됨
- 각 사냥터에는 일반 몬스터 4종이 등장
- 구간의 1~4층은 해당 사냥터 몬스터 4종의 엘리트 버전
- 구간의 5층은 신규 보스
- 엘리트 몬스터는 일반 사냥터에서 얻던 같은 정수를 10% 확률로 드랍
- 보스는 독특한 전투 스타일을 만드는 고유 정수를 드랍
- 탑에서 드랍되는 정수는 1~3등급 중 랜덤이며, 확률은 1등급 50%, 2등급 30%, 3등급 20%

예:

```
사냥터 1 → 탑 1~5층
사냥터 2 → 탑 6~10층
사냥터 3 → 탑 11~15층
```

### 업적 보상

- 현재 업적 보상은 없음
- 보스 최초 클리어 보상은 확정 보스 정수

## 성장 단계

아래는 과거 초안이다.
현재 기준에서 장비 강화와 보석은 추후 업데이트 예정이다.

| 단계 | 내용 | 핵심 활동 |
|------|------|----------|
| 1단계 | 장비 파밍 | 기본 스탯 세팅 |
| 2단계 | 정수 파밍과 합성 | 전투 방식 결정 |
| 3단계 | 보석 파밍과 정수 세팅 | 추후 업데이트 |
| 4단계 | 장비 옵션 정비 | 빌드 완성 |

각 단계는 자연스럽게 해금:
- 탑 일정 층 이상부터 정수 드랍률 상승
- 강화 일정 수준 도달 후 옵션 해금: 추후 업데이트
- 일정 등급 이상 정수에 보석 슬롯 해금: 추후 업데이트

## 스킬 계열 (미확정, 방향만)

메커니즘 기반으로 계열 분류 예정:

딜 계열: 강타, 연격, 마법 공격, 지속 피해(DoT)

강화 계열: 공격 강화, 방어 강화, 속도 강화, 치명타 강화

회복 계열: 즉시 회복, 지속 회복, 흡혈

디버프 계열: 약화, 둔화, 취약

### 연격 특성

장점:
- 회피율 높은 상대에게 유리 (여러 번 굴림)
- 치명타 기댓값 높음 (횟수만큼 굴림)
- 상태이상 스택 빠르게 적재

단점:
- 방어력 높은 상대에게 불리 (분산 피해가 방어력에 많이 깎임)
- 총 피해량이 강타보다 낮을 수 있음
- 단발 버프 효과 활용 불리
