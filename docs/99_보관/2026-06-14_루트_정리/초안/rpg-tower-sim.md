# RPG Tower Simulator

전투 로그를 보면서 몬스터, 정수, 장신구 수치를 조정하기 위한 간단한 CLI/웹 시뮬레이터다.

현재 목적은 실제 게임 코드가 아니라 밸런스 감각 확인이다.
같은 `--seed`를 쓰면 같은 전투 로그가 나오므로 수치 변경 전후 비교에 사용할 수 있다.

## 실행

공용 데이터는 `rpg-tower-data.js`에 둔다.
CLI와 웹 뷰어는 같은 몬스터/정수/장신구 데이터를 읽는다.

```bash
node rpg-tower-sim.mjs list
```

사용 가능한 빌드, 장신구, 몬스터 목록을 출력한다.

```bash
node rpg-tower-sim.mjs duel --build flurry --level 10 --enemy forestWolf --rank elite --enemyLevel 5 --accessory serratedRing --log --seed 7
```

전투 로그가 포함된 1:1 전투를 실행한다.

```bash
node rpg-tower-sim.mjs matrix --enemy forestBoar --seed 3
```

여러 빌드와 일반/엘리트/보스 난이도 결과를 표로 출력한다.

## 주요 옵션

| 옵션 | 설명 |
|------|------|
| `--build` | `str`, `flurry`, `tank`, `mage` |
| `--strategy` | `counter`를 주면 탑 층 상대에 맞춰 빌드/장신구/정수를 자동 선택 |
| `--level` | 플레이어 레벨 |
| `--accessory` | 장신구 키 |
| `--essences` | `wolfFlurry:2,bladeEdge:2,lizardScale:2` 형식 |
| `--enemy` | 몬스터 키 |
| `--rank` | `normal`, `elite`, `boss` |
| `--enemyLevel` | 몬스터 레벨 |
| `--log` | 전투 로그 출력 |
| `--seed` | 난수 고정값 |

## 기존 구현에 반영된 규칙

- 10틱 = 1초
- 공격속도는 `sqrt(공격속도 / 100)`으로 초당 일반공격 게이지를 계산한다.
- 정수 사용은 일반공격 게이지를 소모하지 않는 무료 스킬이다.
- 한 캐릭터는 한 틱에 행동 하나만 수행한다.
- 같은 캐릭터에게 정수와 일반공격이 동시에 준비되어 있으면 정수 슬롯 순서가 일반공격보다 우선한다.
- 일반 몬스터는 전투에서 2단계 정수를 사용하고, 드랍은 1단계 정수다.
- 엘리트와 보스는 전투에서 4단계 정수를 사용하고, 드랍은 1~3단계 정수다.
- 플레이어는 1~5단계 정수를 장착해 비교할 수 있다.
- 물리/마법 피해는 명중과 회피 판정을 받는다.
- 고정 피해, 독, 출혈은 방어와 치명타 영향을 받지 않는다.
- 장신구는 전투 효과 1개를 기준으로 처리한다.
- 장신구 기본 퍼센트 효과는 새 아이템 규칙의 등급별 범위를 참고해 중후반 기준값으로 반영한다.

## 현재 데이터 범위

사냥터 데이터는 `HUNTING_AREAS`에 20개로 정의한다.
각 사냥터는 5레벨 구간을 담당한다.

```txt
1번 사냥터: 1~5레벨
2번 사냥터: 6~10레벨
...
20번 사냥터: 96~100레벨
```

사냥터와 탑 몬스터의 관계:

```txt
각 사냥터 = 탑 5층짜리 구간 1개
구간 1~4층 = 해당 사냥터 몬스터의 elite 버전
구간 5층 = 해당 사냥터 boss
```

같은 몬스터는 사냥터와 탑에서 같은 고유 정수를 공유한다.
예를 들어 사냥터의 성난 멧돼지 normal과 탑의 성난 멧돼지 elite는 모두 `성난 멧돼지의 괴력` 정수에 연결된다.

탑 데이터는 `TOWER_FLOORS`에 1~100층으로 정의한다.
각 층은 층 번호, 사냥터, 몬스터, 등급, 상대 레벨, 목표 문구를 가진다.

탑 등급 규칙:

```txt
각 5층 구간의 1~4층: 엘리트
각 5층 구간의 5층: 보스
```

탑 레벨 스케일:

```txt
1~10층: 상대 레벨 = 층
11~30층: 상대 레벨 = 10 + ceil((층 - 10) × 1.5)
31~50층: 상대 레벨 = 40 + ceil((층 - 30) × 2)
51~60층: 상대 레벨 = 80 + ceil((층 - 50) × 2)
61~100층: 상대 레벨 = 100 + ceil((층 - 60) × 2.5)

10층: 10레벨
30층: 40레벨
50층: 80레벨
60층: 100레벨 권장선
100층: 200레벨급 엔드 콘텐츠
```

각 몬스터는 고유 정수 1개를 가진다.
정수는 1~5단계 데이터를 가지며, 시뮬레이터에서는 현재 엔진이 처리할 수 있는 피해, 버프, 회복, 지속 피해, 상태이상, 명중/회피, 치명타, 반사, 쿨타임 환급 효과 위주로 구현했다.

## 해석 기준

| 대상 | 클리어 조건 |
|------|-------------|
| 일반 사냥 몬스터 | 권장 레벨 + 평균 장비로 안정적으로 처치 가능 |
| 탑 1~60층 | 100레벨 기본 빌드가 60층 전후까지 진행 가능 |
| 탑 61~100층 | 100레벨 이후 장비/정수 성장으로 미는 엔드 콘텐츠 |

- 전투 시간은 결과로 관찰하되 목표로 삼지 않는다.
- 권장 레벨보다 낮으면 실패하고, 맞추면 클리어되는 것이 기준이다.

시뮬레이터의 기본 장비는 “해당 레벨에서 적당히 맞춘 장비”를 가정한다.
장신구와 정수는 전투 시간과 안정성을 바꾸는 변수로 본다.

현재 기준 체크:

```bash
node rpg-tower-sim.mjs tower --build str --level 100 --from 55 --to 70 --seed 7
```

기본 근력 빌드, 장신구 없음 기준으로 60층 보스는 클리어하고 75층부터 첫 실패가 나온다.

맞춤 전략 체크:

```bash
node rpg-tower-sim.mjs tower --strategy counter --level 100 --from 64 --to 75 --seed 7
```

상대 성향에 따라 다음 세팅 중 하나를 고른다.

| 전략 | 용도 |
|------|------|
| 기본 | 특별한 위협이 낮은 상대 |
| 대마법 | 지력/지혜가 높은 마법형 상대 |
| 흡혈 돌파 | 기사, 야수, 보스, 강한 물리 압박 상대 |
| 명중 연격 | 회피/손재주가 높은 상대 |
| 파쇄 마법 | 방어/체력이 높은 상대 |
| 생존 | 장기전 생존 확인용 |

현재 100레벨 기준으로 기본 근력 고정은 75층에서 첫 실패가 나온다.
맞춤 전략은 상대별로 다른 세팅을 고르며, 기본 근력으로 막히는 일부 층을 통과한다.
예를 들어 74층 유리 독사는 명중/생존 문제 때문에 기본 전략과 다른 흡혈 돌파 세팅을 선택한다.
