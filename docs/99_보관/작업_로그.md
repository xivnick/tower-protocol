## 이전 결정 기록

- 일반 몬스터는 전투에서 2등급 정수를 사용하고, 처치 시 1등급 정수를 드랍한다.
- 엘리트 몬스터와 보스 몬스터는 전투에서 4등급 정수를 사용하고, 처치 시 1~3등급 정수를 드랍한다.
- 탑은 `1~4층 엘리트 + 5층 보스` 구간을 반복한다. 100층은 20개 구간으로 구성된다.
- 사냥터는 20개가 맞으며, 각 사냥터는 탑 5층짜리 구간 1개와 연결된다.
- `rpg-tower-data.js`는 20개 사냥터 각각에 일반 몬스터 4종과 보스 1종을 가진다.
- UI 예시 데이터는 버리고 실제 데이터 이름을 기준으로 한다.
- 보석은 보류 기능이며 추후 업데이트 예정이다.
- 인증은 필요하지만 최대한 간결하게 구현한다.
- 장비 강화는 추후 업데이트 예정이다.
- 장비에는 레벨 제한이 없다. 다만 드랍 구간은 사냥터에 맞춘다.
- 탑 입장 비용은 실제 게임 규칙이다.
- 보스 최초 클리어 보상은 확정 정수다.
- 업적 보상은 현재 없다.
- 휴식: 자동사냥 전투 종료 후 설정에 따라 `RECOVERY_ACTION_SECONDS`만큼 시간을 소모하며, 휴식 후 체력은 항상 100%가 된다.
- 휴식 중 액티브 정수 쿨타임도 `RECOVERY_ACTION_SECONDS`만큼 감소한다.
- 휴식 실행 시점에 포션을 보유 중이면 포션을 자동 사용한다.
- 포션 사용은 휴식을 한 것으로 취급한다. 포션 사용 시에도 `RECOVERY_ACTION_SECONDS`만큼 시간이 흐르고, 액티브 정수 쿨타임도 같은 값만큼 감소한다.
- 배포 후 밸런스 데이터는 어드민에서 개별 entity로 편집하고, publish 시 compiled snapshot을 생성해 active version으로 반영한다.
